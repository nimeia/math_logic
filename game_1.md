下面我按 **简单 / 中等 / 困难** 三档，每档给出一张“题目模板表”——每行就是一个可以直接在程序里实现的出题规则。
每个模板都强调：**起始数字和参数要随机**，以防孩子背题不学规律。

> 说明：
>
> * `a1` 表示首项
> * `n` 表示第 n 项
> * 你后端可以把“参数与生成规则”这一列直接翻译成代码逻辑

## 一、简单难度（L1）模板表

> 特征：只用加减 / ×2 / ÷2，非常直观；
> 数值尽量控制在 **0～50**；只缺 **最后一项** 或 **某一中间项**。

| 模板ID | 规律描述                          | 参数与生成规则                                                                                         | 出题形式                           | 随机化建议                                        | 示例（仅示意）                                             |
| ---- | ----------------------------- | ----------------------------------------------------------------------------------------------- | ------------------------------ | -------------------------------------------- | --------------------------------------------------- |
| L1-1 | **等差递增：每次加同一个数**              | 设公差 `d`，`d ∈ {1,2,3,4,5}`；随机选 `a1 ∈ [0, 30]`，生成 `a_n = a1 + (n-1)*d`，n=1..5                     | 常用：给出前4项，问第5项；也可以给出5项，挖空第3或第4项 | `a1` 和 `d` 每次随机；可给 `d` 小权重区分难易（如 d=1、2 出现更多） | 如：**7, 9, 11, 13, ( ? )**（d=2）                      |
| L1-2 | **等差递减：每次减同一个数**              | 设步长 `d ∈ {1,2,3,4,5}`；随机选 `a1 ∈ [10, 50]`，保证所有项非负：`a_n = a1 - (n-1)*d`                          | 同上：优先缺最后一项；偶尔缺中间项              | `a1` 必须 ≥ `d*(项数-1)`；每次随机选 `a1` 和 `d`        | 如：**20, 17, 14, 11, ( ? )**（d=3）                    |
| L1-3 | **整十数列：按 10 为单位数数**（特殊等差）     | 固定 `d = 10`；随机 `a1 ∈ {0, 10, 20}`；`a_n = a1 + (n-1)*10`                                         | 推荐缺最后一项，让孩子感受“十进制跳数”           | `a1` 每次随机；不要固定从 10 开始                        | 如：**10, 20, 30, 40, ( ? )**                         |
| L1-4 | **简单等比 ×2**                   | 设公比 `q = 2`；随机选 `a1 ∈ {1,2,3,4,5}`；`a_n = a1 * 2^(n-1)` 且结果 ≤ 100                               | 通常给出前4项，问第5项                   | `a1` 每次随机；可控制结果最大值（超限则重选）                    | 如：**3, 6, 12, 24, ( ? )**                           |
| L1-5 | **简单等比 ÷2（或 ×1/2）**           | 从一个 2 的幂开始：随机选 `k ∈ {3,4,5,6}`，`a1 = 2^k`；`a_n = a1 / 2^(n-1)`（保持整数）                            | 给出前4项，问第5项；或给出5项挖空中间           | `k` 每次随机；可避免总是 16, 8, 4… 那一组                 | 如：**32, 16, 8, 4, ( ? )**                           |
| L1-6 | **差值递增：+1, +2, +3,... 的“台阶”** | 设首项 `a1 ∈ [0, 10]`；差值从1开始逐次递增：`d1=1, d2=2, d3=3,...`，`a_(n) = a_(n-1) + dn-1`                   | 给出前5项，问第6项；或给出6项挖空第3/4项        | `a1` 每次随机；项数不宜太多（5~6项即可）                     | 如：**1, 2, 4, 7, 11, ( ? )**（+1,+2,+3,+4,+5）         |
| L1-7 | **简单周期性加减：+a, +a, +b 重复**     | 设 `a ∈ {1,2,3}`, `b ∈ {4,5,6}` 且 b > a；随机 `a1 ∈ [0, 20]`；周期差值为 `[+a, +a, +b]` 循环                | 给出前5~6项，问下一项；暂不挖中间             | 每次随机 `a1, a, b`；但约束总结果 ≤ 100                 | 如（a=1,b=3）：**2, 3, 4, 7, 8, 9, ( ? )**（+1,+1,+3 循环） |
| L1-8 | **最简单奇偶位不同：奇数位等差，偶数位常数**      | 随机 `a1 ∈ [1, 20]`，公差 `d ∈ {1,2,3}`；设偶数位固定为某小数 `c ∈ [1,9]`；奇数位：`a_(2k-1) = a1 + (k-1)*d`，偶数位 = c | 给出5项，问第6项（多用后期的“简单难度尾部关卡”）     | `a1, d, c` 每次随机；注意不要让奇偶看起来像普通等差              | 如：**3, 10, 5, 10, 7, ( ? )**（奇数位3,5,7,…；偶数位一直10）    |

## 二、中等难度（L2）模板表

> 特征：出现**变差**、**交替/双序列**、**平方**、**基础斐波那契**；
> 数值约在 **0～200**，主要用加减 + 小乘除。

| 模板ID | 规律描述                          | 参数与生成规则                                                                                                                     | 出题形式                        | 随机化建议                                  | 示例（仅示意）                                              |
| ---- | ----------------------------- | --------------------------------------------------------------------------------------------------------------------------- | --------------------------- | -------------------------------------- | ---------------------------------------------------- |
| L2-1 | **变差数列：差值是等差数列（+3,+5,+7,…）**  | 随机选 `a1 ∈ [0, 20]`；随机选初始差 `d0 ∈ {1,2,3}`；差值序列：`d1=d0, d2=d0+2, d3=d0+4, ...`；`a_n = a_(n-1) + dn-1`                         | 生成6项，给出前5项问第6项；或6项挖空第4      | `a1, d0` 每次随机；控制 `a_n ≤ 200`           | 如：**2, 5, 10, 17, 26, ( ? )**（差为+3,+5,+7,+9,+11）     |
| L2-2 | **奇偶位双等差：奇数位一条等差，偶数位另一条等差**   | 随机 `a1 ∈ [0,20]`、`d1 ∈ {1,2,3}`；随机 `b1 ∈ [0,20]`、`d2 ∈ {1,2,3}`；奇数位：`a_(2k-1) = a1 + (k-1)*d1`；偶数位：`a_(2k) = b1 + (k-1)*d2` | 生成6项；通常缺最后一项，也可缺第3或第4项      | 每局随机 a1,b1,d1,d2；避免两条序列完全一样（否则退化成简单等差） | 如：**2, 10, 4, 12, 6, 14, ( ? )**（奇：2,4,6；偶：10,12,14） |
| L2-3 | **整齐平方数列：1²,2²,3²…**          | 随机选择起始指数 `k ∈ {1,2,3}`；`a_n = (k + n - 1)^2`，但强约束 a_n ≤ 200                                                                 | 生成5~6项，缺最后一项                | 每次随机 `k`；避免总从1²开始                      | 如：**4, 9, 16, 25, 36, ( ? )**（从2²开始）                 |
| L2-4 | **平方数列变形：n² ± 常数C**           | 随机 `k ∈ {1,2,3}`；随机 `C ∈ {-3,-2,-1,1,2,3}`；`base_n = (k+n-1)^2`，`a_n = base_n + C`，结果控制在 0~200                              | 生成5项，缺最后一项；或5项挖空第3          | `k` 和 `C` 每次随机；防止结果为负                  | 如：**2, 7, 14, 23, 34, ( ? )**（等于 n²+1 的变形）           |
| L2-5 | **标准斐波那契型：aₙ = aₙ₋₁ + aₙ₋₂**  | 随机选两个小起始值 `a1,a2 ∈ {1..9}`，但避免太大（比如不超过5），然后按 `a_n = a_(n-1)+a_(n-2)` 生成，限制 `a_n ≤ 200`                                      | 生成6项，缺最后一项；可在较高关卡缺中间一项      | 每局随机 `a1,a2`；避免总是(1,1)                 | 如：**2, 3, 5, 8, 13, 21, ( ? )**                      |
| L2-6 | **固定周期交替：(+a,+b,+a,+b,...)**  | 随机 `a ∈ {1,2,3}`, `b ∈ {4,5,6}`；随机 `a1 ∈ [0,30]`；差值周期 `[+a,+b]` 循环                                                          | 生成6项，缺最后一项；或挖空一个“b”对应的位置    | `a1,a,b` 每次随机；控制最高值≤200                | 如：**3, 5, 9, 11, 15, 17, ( ? )**（+2,+4 循环）           |
| L2-7 | **“前两项和 + 常数C” 递推**           | 随机 `a1,a2 ∈ {1..9}`，随机 `C ∈ {-3,-2,-1,1,2,3}`；递推：`a_n = a_(n-1) + a_(n-2) + C`，控制值≤200                                      | 生成6项，缺最后一项；面向中等难度后半段        | 每次随机 a1,a2,C；如果生成时超范围则重选               | 如：**1, 2, 4, 7, 13, 24, ( ? )**（≈前两项和+1 的一类）         |
| L2-8 | **乘加型简单递推：aₙ = aₙ₋₁ × k + C** | 随机 `a1 ∈ {1..9}`；`k ∈ {2,3}`, `C ∈ {1,2,3,-1}`；递推 `a_n = a_(n-1)*k + C`，控制≤200                                              | 常用：生成5项，缺第5项；适合作为“中等-难一点”的题 | 参数随机关联；注意避免结果太快爆炸                      | 如：**2, 5, 11, 23, ( ? )**（×2+1）                      |

## 三、困难难度（L3）模板表

> 特征：组合、交叉、递推更复杂，一题可能涉及两步以上推理；
> 数值控制在 **≤1000**，但尽量少超 3 位数。

| 模板ID | 规律描述                              | 参数与生成规则                                                                                                                           | 出题形式                           | 随机化建议                                    | 示例（仅示意）                                                                     |
| ---- | --------------------------------- | --------------------------------------------------------------------------------------------------------------------------------- | ------------------------------ | ---------------------------------------- | --------------------------------------------------------------------------- |
| L3-1 | **复杂变差：差值本身是等差，且起始差较大**           | 随机 `a1 ∈ [0,50]`；随机 `d0 ∈ {2,3,4,5}`，增量 `step ∈ {1,2}`；差值序列：`d1=d0, d2=d0+step, d3=d0+2*step,...`；`a_n = a_(n-1) + dn-1`，控制≤1000  | 生成7项，缺第6或第7项                   | `a1,d0,step` 每次随机；可限制项数=6~7              | 如：**3, 7, 12, 18, 25, 33, ( ? )**                                           |
| L3-2 | **双数列交叉：奇数位等差，偶数位等比**             | 随机：奇数序列 `x1 ∈ [1,30], dx ∈ {1..5}`；偶数序列 `y1 ∈ {1,2,3,4}, q ∈ {2,3}`；定义：`a_(2k-1) = x1+(k-1)*dx`；`a_(2k) = y1 * q^(k-1)`；控制整体≤1000 | 生成7项，常缺最后一项；也可缺某偶数位            | 每次随机上述4个参数；限制偶数位别太快爆表                    | 如：**5, 3, 10, 6, 15, 12, 20, ( ? )**                                        |
| L3-3 | **双数列交叉：两条等差但差不同，起点不同**           | 随机：奇数: `x1, dx`；偶数: `y1, dy`，都在小范围整数内；同 L3-2，只是两条都是等差                                                                             | 生成7项；缺中间或最后一项                  | a1,b1,dx,dy 随机；避免 dx=dy 且 x1≈y1（否则太像简单题） | 如：**2, 9, 5, 12, 8, 15, 11, ( ? )**                                         |
| L3-4 | **斐波那契变体：aₙ = aₙ₋₁ + aₙ₋₂ + C**   | 随机 `a1,a2 ∈ {1..9}`；随机 `C ∈ {-5..5}`（不含0也可以）；按此递推，限制≤1000                                                                         | 生成7项，缺第6或第7项                   | a1,a2,C 每次随机；生成时要检测溢出                    | 如：**1, 4, 8, 13, 25, 46, ( ? )**（示例类型）                                      |
| L3-5 | **二阶等差 / 类二次数列：像 n² 但平移或拉伸**      | 随机选择：`a,b,c` 构成 `a_n = an² + bn + c` 但参数要小（如 a∈{1,2}, b,c 在小整数）；通过算出前7项作为数列                                                       | 生成7项，缺第5或第7项                   | 可用固定少量的 (a,b,c) 组合 + 随机平移起始 n；避免过难       | 如：**2, 5, 10, 17, 26, 37, 50, ( ? )**                                       |
| L3-6 | **乘加型递推：aₙ = (aₙ₋₁ × k) ± m，k>1** | 随机 `a1 ∈ {1..9}`；`k ∈ {2,3}`, `m ∈ {1..5}`，递推 `a_n = a_(n-1)*k +/− m`，控制≤1000                                                     | 生成6~7项，缺中间或最后                  | 多试几次防止太快超 1000；参数每次随机                    | 如：**2, 5, 11, 23, 47, ( ? )**（×2+1型）                                        |
| L3-7 | **交替规则：一位做等差，下一位做乘法（+a, ×k 交替）**  | 随机 `a1 ∈ {1..9}`；`add ∈ {1..5}`, `k ∈ {2,3}`；按 `[+add, ×k]` 周期递推：`a2=a1+add; a3=a2*k; a4=a3+add; a5=a4*k...`，控制≤1000              | 生成7项，缺最后一项；也可缺内层进行挑战           | `a1,add,k` 每次随机；检测数值是否过大                 | 如：**3, 6, 12, 15, 30, 33, 66, ( ? )**                                       |

## 四、关于“随机初始化”的统一建议

你可以在技术实现上统一几条“反背题”策略：

1. **模板驱动出题，而不是固定题库**

   * 每道题由“模板 + 随机参数”生成，避免题目固定不变。
   * 同一个模板，每次随机 `a1`、公差、公比、常数 C 等参数。

2. **限制结果范围 + 重抽机制**

   * 如果生成数列中出现负数、超过上限（比如 1000）或非整数，就**丢弃**这组参数重新随机一次。

3. **随机缺失位置（在规则允许范围内）**

   * 对于同一模板，可以随机决定缺**最后一项**或某个**中间项**（受难度限制：L1 尽量缺最后，L2/L3 可缺中间）。
   * 这样即使同一参数重复出现，问号位置也不同，孩子不能靠记忆直接回答。

4. **可以记录近期生成题目参数，避免短时间重复**

   * 例如保留最近 N 道题的 `(模板ID, 参数集合)`，新题生成时尽量避开完全相同组合。
